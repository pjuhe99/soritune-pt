# PT 매칭 캘린더 — 예정일 입력을 클릭형 캘린더로 교체

**Date:** 2026-05-12
**Status:** Approved, ready for plan
**Owner:** pt-dev

## 배경

PT 매칭 캘린더(`/admin/#coaching-calendar`)의 생성/수정 모달은 현재 예정일을 **textarea**(한 줄에 하나, `YYYY-MM-DD`)로 받는다. 자동 패턴 생성 버튼이 후보를 채워주면 사용자가 직접 텍스트를 편집한다.

사용자가 보고한 문제: 텍스트 입력이 직관성이 떨어지고, 특정 날짜만 빼거나 옮길 때 캘린더를 보면서 작업하고 싶다.

## 목표

예정일 입력 UI를 **클릭형 월 그리드 캘린더**로 완전 대체. 텍스트 입력은 제거. 자동 패턴 버튼은 유지하되 결과를 캘린더 선택 상태로 채운다.

## Non-goals

- 서버 API 스키마 변경 없음 (POST body의 `dates: ["YYYY-MM-DD", ...]` 그대로).
- 자동 패턴 후보 생성 로직 변경 없음 (현 endpoint 그대로 호출).
- 모바일 전용 별도 UI는 만들지 않음 (그리드가 자연스럽게 반응형).
- 캘린더 전반 UX 재설계(스타일 시스템 통합 등) — 이번 작업 범위 밖.

## 디자인

### 모달 구조

생성/수정 모달 안에서 textarea 영역만 캘린더 영역으로 교체. 그 외(매칭월, 상품명, 회차 수, 시작일, 패턴, 자동 패턴 버튼)는 기존 그대로.

```
┌─ 매칭월 / 상품명 / 회차 수 / 시작일 / 패턴 (기존 그대로) ┐
│                                                     │
│  [자동 패턴 생성] [전체 해제]   |  회차 5 / 선택 3 ← 일치=초록, 불일치=빨강
│                                                     │
│           ←   May 2026   →    ← 좌우 화살표 + 월 라벨   │
│   일 월 화 수 목 금 토                                │
│  ─────────────────────                              │
│   .  .  .  .  1  2  3       ← 셀 (오늘=테두리, 선택=오렌지, 과거=흐림)│
│   4  5  6  7  8  9 10                                │
│  ...                                                │
└─────────────────────────────────────────────────────┘
```

### 셀 상태

| 상태 | 시각 표현 |
|---|---|
| 미선택 + 미래/오늘 | 기본 배경, hover 강조 |
| 미선택 + 과거 (today 이전) | muted (글자색 회색) — 클릭 가능 |
| 오늘 | `outline: 2px solid var(--accent)` 추가 |
| 선택됨 | PT 오렌지(`#FF5E00`) 배경 + 흰 글씨 |
| 다른 달 spillover (해당 월 외 날짜) | 옅은 회색, 클릭 비활성 |

### 상호작용

| 동작 | 결과 |
|---|---|
| 셀 클릭 | 선택 `Set<YYYY-MM-DD>` toggle. 다시 클릭하면 해제. |
| 좌/우 화살표 | `currentMonth` view만 변경. 선택은 유지. |
| 자동 패턴 생성 버튼 | 서버에서 후보 받기 → 선택 set을 결과로 **replace**(기존 다 지움) → view를 첫 날짜의 달로 점프. |
| 전체 해제 버튼 | 선택 set 비움. view 유지. |
| 저장 | `Array.from(selectedSet).sort()`로 dates 배열 만들어 기존 POST body 그대로. |

### 시작 달 결정 (모달 open 시점)

- **신규 생성**: cohort_month의 1일 기준 (`YYYY-MM-01`).
- **수정**: `dates[0]`의 달. (현재 데이터 모델상 수정 모달에서 dates가 빈 경우는 없지만, 방어적으로 비어있으면 cohort_month fallback.)

### 상단 카운트 배지

- 라벨: `회차 N / 선택 M`
- 색: `N === M` → 초록 (`--positive`), 불일치 → 빨강 (`--negative`).
- 위치: 자동 패턴 / 전체 해제 버튼 우측, 캘린더 헤더 위.

### 검증

- 신규: 선택 수 = 회차 수 필수. 저장 시 불일치면 기존 alert 그대로.
- 수정: 선택 수 = 회차 수 필수. 비어 있으면(전체 해제 후 저장 시도) "선택 후 저장" alert.
- 회차 수를 사용자가 변경하면 배지만 즉시 빨강/초록으로 갱신 (선택은 건드리지 않음).

## 컴포넌트 경계

캘린더 UI는 `coaching-calendar.js` 내부 모듈로 한정한다. 외부에서 공유하지 않으므로 별도 파일로 분리하지 않는다. 다음 헬퍼 함수만 page 객체 안에 추가:

| 함수 | 역할 |
|---|---|
| `_buildMonthGrid(year, month)` | 주어진 달의 7×6 그리드 cell 배열 반환. 각 cell은 `{date: 'YYYY-MM-DD' \| null, inMonth: bool, isToday: bool, isPast: bool}`. |
| `_renderCalendar()` | 현재 `_selectedDates` Set + `_viewMonth` 기준으로 캘린더 HTML 재렌더링. |
| `_toggleDate(dateStr)` | 선택 set 토글 + 배지 갱신 + 해당 셀만 클래스 갱신. |
| `_setSelection(datesArray)` | 자동 패턴 결과를 받아 set replace + 첫 날짜 달로 view 이동 + 전체 재렌더링. |
| `_updateBadge()` | `회차 N / 선택 M` 배지 색/숫자 갱신. |

선택 상태 저장은 page 객체 인스턴스의 `_selectedDates: Set<string>`, view 상태는 `_viewMonth: {year, month}`. 모달 close 시 모두 해제.

## 서버 영향

**없음.** POST body의 `dates` 배열 형식이 동일하고, 자동 패턴 endpoint(`/api/coaching_calendar.php?action=auto_pattern`)도 동일 호출. 기존 PHP 테스트 회귀에 영향 없음.

## 파일 변경

| 파일 | 변경 |
|---|---|
| `public_html/admin/js/pages/coaching-calendar.js` | textarea 영역 제거 + 캘린더 그리드/헬퍼 함수 추가. 예상 +120 / -15 줄. |

CSS는 inline style 위주 (PT 어드민 다른 페이지와 동일 패턴). 새 CSS 클래스 추가는 최소화.

## 테스트

PT는 PHP 테스트 인프라만 있고 JS 테스트는 없으므로 다음으로 검증:

1. **회귀**: `php tests/run_tests.php` → 646/646 PASS 유지 (서버 변경 0이므로 자동).
2. **DEV 브라우저 smoke**:
   - 신규: 시작일+패턴으로 자동 패턴 → 선택 표시 → 일부 토글 → 저장 → list 갱신.
   - 수정: 모달 열면 기존 dates가 선택 표시됨 → 토글 조정 → 저장.
   - 전체 해제 → 저장 시도 → "선택 후 저장" alert.
   - 좌우 화살표로 cohort 걸치는 달 이동 후 추가 선택.
   - 과거 날짜 클릭 (수정 시 과거 회차 조정 케이스).
